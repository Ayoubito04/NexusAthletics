================================================================================
                          TESTING DE SEGURIDAD
================================================================================

1. VERIFICAR HELMET HEADERS:
   $ curl -i http://localhost:3000/

   Debe incluir:
   ✓ Strict-Transport-Security: max-age=31536000
   ✓ Content-Security-Policy: default-src 'self'
   ✓ X-Frame-Options: DENY
   ✓ X-Content-Type-Options: nosniff
   ✗ X-Powered-By: (debe estar ausente)

2. VERIFICAR RATE LIMITING:
   $ for i in {1..6}; do curl -X POST http://localhost:3000/auth/login \
     -H "Content-Type: application/json" \
     -d '{"email":"test@test.com","password":"wrong"}' \
     -w "\nStatus: %{http_code}\n"; done

   Expected: 200 200 200 200 200 429
   (5 intentos exitosos, el 6to rechazado por rate limit)

3. VERIFICAR JWT ALGORITMO:
   $ node -e "
   const jwt = require('jsonwebtoken');
   const secret = 'test-secret-64-chars';
   const token = jwt.sign({id:1}, secret);
   jwt.verify(token, secret, {algorithms:['HS256']}, (err) => {
     if (err) console.log('❌ ERROR:', err.message);
     else console.log('✅ JWT HS256 verified correctly');
   });
   "

4. VERIFICAR CORS REJECT:
   $ curl -H "Origin: http://evil.com" http://localhost:3000/ -v

   Debe retornar 403 o sin header CORS

================================================================================
                          RIESGOS RESIDUALES
================================================================================

VULNERABILIDADES NO ABORDADAS (Requieren auditoría posterior):

Riesgo Alto:
  • SQL Injection - Requiere auditoría de controllers
  • Broken Authentication - Sin refresh token rotation
  • Sensitive Data Exposure - Sin cifrado de PII en BD
  • Admin Routes - Verificar isAdmin en TODAS rutas /admin

Riesgo Medio:
  • Password Hashing - Verificar bcrypt cost >= 10
  • HTTPS Enforcement - Implementar en proxy (no app-level)
  • Secrets Rotation - Manual cada 6 meses

Riesgo Bajo:
  • Input Validation - express-validator instalado, pendiente integración
  • CORS Configuration - Solo si se agregan nuevos origins
  • Rate Limiting Bypass - Con múltiples workers, usar redis-rate-limit
